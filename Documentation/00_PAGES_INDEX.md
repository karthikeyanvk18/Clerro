## Mobile Responsive Design

All pages are fully responsive with:
- **Mobile Navigation**: Bottom navigation bar
- **Mobile Header**: Simplified top bar
- **Mobile Layout**: Single column, optimized scrolling
- **Mobile Inputs**: Large touch targets
- **Mobile Charts**: Responsive sizing

---

## Common UI Elements

Across all pages:
- **Sidebar**: Main navigation (hidden on mobile)
- **Top Header**: Logo, search, user menu
- **Bottom Nav**: Quick access to main sections (mobile)
- **Cards**: Information containers with actions
- **Buttons**: Primary, secondary, outline variants
- **Icons**: Lucide React icons throughout
- **Charts**: Recharts visualizations
- **Forms**: Input fields, selects, date pickers
- **Modals**: Dialogs for actions
- **Toast**: Notifications for feedback

## Navigation Patterns

1. **Sidebar Navigation** (Desktop): Left sidebar with main menu
2. **Bottom Navigation** (Mobile): Tab-like navigation at bottom
3. **Breadcrumb**: Not typically used, back button instead
4. **Drawer**: Side drawer for filters/options
5. **Tabs**: Horizontal tabs for content switching
6. **Dropdown**: Select menus for sorting/filtering

---

## Data Display Patterns

1. **Cards**: Primary data containers
2. **Lists**: Scrollable content lists
3. **Tables**: Tabular data (responsive on mobile)
4. **Charts**: Various Recharts visualizations
5. **Modals**: Detailed views and forms
6. **Sheets**: Bottom sheets (mobile-first)
7. **Toast**: Transient notifications

---

## Interaction Patterns

1. **Click/Tap**: Primary interaction
2. **Swipe**: Slide navigation (mobile)
3. **Long Press**: Context menu (mobile)
4. **Pull-to-Refresh**: Reload data (mobile)
5. **Infinite Scroll**: Load more content
6. **Search**: Real-time filtering
7. **Filter/Sort**: Dynamic list updates

---

## Performance Considerations

- **Code Splitting**: Lazy loading by page
- **Image Optimization**: Compressed assets
- **Bundle Size**: Tree-shaking unused code
- **Caching**: LocalStorage for user data
- **API Calls**: Optimized queries, pagination
- **Render**: React.memo for expensive components
